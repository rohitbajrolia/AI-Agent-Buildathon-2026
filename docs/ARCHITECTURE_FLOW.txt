HOME INSURANCE MCP - ARCHITECTURE AND EXECUTION FLOW

Purpose
This project is a local, document-grounded Q&A workflow for homeowners insurance.
It combines Streamlit (UI), LangGraph (flow control), MCP (tool protocol),
Qdrant (vector search), and OpenAI (embeddings + answer generation).

What each technology does in this repo
1) Streamlit: renders the UI and calls LangGraph when the user clicks Ask.
   - home-insurance-mcp-client/src/client/app.py:308
2) LangGraph: orchestrates the steps retrieve -> validate -> answer -> citation_verify.
   - home-insurance-mcp-client/src/client/graph.py:682
3) MCP: a tool protocol; the client calls tools on the server over streamable HTTP.
   - home-insurance-mcp-client/src/client/mcp_client.py:22
   - home-insurance-mcp/src/server/mcp_server.py:290
4) Qdrant: stores embeddings and returns nearest chunks for retrieval.
   - home-insurance-mcp/src/server/mcp_server.py:268
5) OpenAI: embeddings (indexing/retrieval) and chat completions (answer + rewrite).
   - home-insurance-mcp/src/server/mcp_server.py:268
   - home-insurance-mcp-client/src/client/graph.py:242
